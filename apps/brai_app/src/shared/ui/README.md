# Shared UI Foundation

This folder is the Brai source-owned shadcn-compatible UI layer.

- `cn.ts` and primitive components are local code, not a black-box UI kit.
- Use `components.json` as the shadcn CLI source of truth for `new-york`, neutral base color, CSS variables, aliases, registries, and Lucide icons; Tailwind v4 keeps `tailwind.config` blank and uses `src/app/globals.css`.
- Build UI from existing local default shadcn primitives, official `@shadcn` components/blocks, or approved free/open shadcn-compatible registry source by default.
- Do not add agent-selected colors, fonts, radii, shadows, gradients, motifs, or custom visual recipes by default. Add custom visual decisions only when the project owner explicitly asks for them.
- Product colors use semantic shadcn/Tailwind tokens such as `bg-background`, `bg-card`, `text-foreground`, `text-muted-foreground`, `border-border`, `border-input`, `ring-ring`, `bg-primary`, `text-primary-foreground`, `bg-muted`, and `bg-accent`. Do not add hardcoded hex/RGB/OKLCH/color-mix values or arbitrary color utilities in product UI.
- Radius, shadow, spacing, and typography use standard Tailwind/shadcn utility names or copied accepted source. Do not add static `rounded-[...]`, `shadow-[...]`, `font-[...]`, new font families, or a separate local visual scale by default.
- Runtime theme controls stay inside standard shadcn light/dark token modes. Do not add arbitrary accent/color pickers unless the project owner explicitly requests that product capability.
- Use the source-owned COSS-compatible `card.tsx`, `table.tsx`, and `badge.tsx` primitives when copying COSS table/card/badge blocks. Bare Tailwind `border` and `outline` utilities are normalized globally in `src/app/globals.css`; do not patch copied components with local border-color hacks.
- Use the source-owned shadcn `label.tsx`, `field.tsx`, `separator.tsx`, and `progress.tsx` primitives for labeled progress and form-style rows. `switch.tsx` remains source-owned COSS-compatible.
- Record source, free/pro status, dependencies, reason, and final ownership before keeping imported blocks.
- Use Radix UI Primitives as the default accessibility and behavior substrate for complex interactive primitives such as dialogs, menus, popovers, tabs, tooltips, checkboxes, switches, and sliders.
- Keep Radix usage behind source-owned wrappers or approved shadcn-compatible blocks in product code. If a file imports a specific `@radix-ui/react-*` package directly, keep that package declared as a direct dependency; the installed `radix-ui` barrel is for sources that intentionally import from `radix-ui`.
- Do not add Radix Themes or another design-system provider unless the project owner explicitly approves it.
- Visible product scrolling uses the source-owned `ScrollArea` wrapper. Its global standard owns the scrollbar width, content gutter, edge inset, and 1000ms hide-after-scroll behavior; product screens must not add local scrollbar gap classes such as `pr-5`, `right-1`, or per-panel scrollbar positioning. Mobile context sheets render it full-width with `contentInset="balanced"` instead of sheet-level horizontal padding. Use `scrollbar={false}` only for non-content layout scroll containers or explicitly approved visual exceptions such as the Focus timer pane; it must hide only the visual scrollbar chrome, not disable native wheel/touch scrolling in the viewport.
- Typography defaults to Geist Sans for app UI and Geist Mono for monospace through the project font variables in `src/app/globals.css`.
- Product UI uses standard Tailwind/shadcn typography utility names such as `font-normal`, `font-medium`, and `font-semibold`, plus source component defaults. Do not revive the old 200/400/600-only rule unless the project owner explicitly asks for that custom type scale.
- Product UI font sizes use standard Tailwind/shadcn utilities only: `text-xs`, `text-sm`, `text-base`, `text-lg`, `text-xl`, and `text-2xl` for normal UI; `text-5xl` and larger only for explicit major metrics such as the timer or primary goal progress. Do not add `text-[...]`, viewport-scaled type, local CSS `font-size`, or a separate Brai type scale by default.

- Configured registries live in `components.json`: `@magicui`, `@shadcnblocks`, `@motion-primitives`, `@coss`, `@shadcnspace`, `@react-bits`, and `@aceternity`.
- Allowed defaults: official shadcn/ui, Radix Primitives, Motion Primitives, shadcnblocks free, Magic UI free, coss free components, ShadcnSpace free items, React Bits free items with mobile/performance review, Aceternity free/source-accessible items, and Tremor Raw/free through manual source review when analytics needs it.
- Tremor is not a configured registry until its shadcn registry endpoint is confirmed.
- Do not add paid, Pro, All Access, API-key gated, private-token, or non-free blocks.
- Do not use product screens directly from `imported`; move reusable neutral code into `blocks` or Brai-specific code into `bright`.
- When the project owner asks to reproduce a selected component or block, copy the actual accepted source as the baseline. Do not rebuild it from a screenshot, description, registry metadata, memory, or a visually similar substitute.
- Exact reproduction preserves source structure, classes, spacing, typography, radii, colors, animation, responsive behavior, and interactions. Only import/path compatibility, replacing source demo content with existing Brai content, explicitly requested content/copy/image/icon changes, data wiring, Brai actions, explicitly requested removals, and necessary size constraints may change.
- If the project owner asks to replace an existing Brai area and does not request content changes, preserve existing Brai content, data bindings, order, meaning, visibility, and item count. Demo content from the copied source is only slot guidance.
- If the source cannot be accessed or copied, treat that as a blocker and ask the project owner. Do not create a custom fallback.
- Design-skill mockups, preview HTML, and generated design-system notes are references only; implementation still comes through this source-owned shadcn-compatible layer unless the project owner explicitly approves another source of truth.
- Prefer `motion` / `motion/react` for animation. Get explicit approval before heavier engines or canvas/shader-heavy effects enter product screens.
- Use `lucide-react` SVG icons by default for controls, navigation, and product UI. Emoji are content/tone choices, not the default app icon system.

## Imported Registry Components

- `EvilEye/EvilEye.tsx` comes from `@react-bits/EvilEye-TS-TW` (`https://reactbits.dev/r/EvilEye-TS-TW.json`). Status: public/free registry item, no Pro/paid/gated marker in registry metadata. Dependency: `ogl@^1.0.11`. Usage: Focus background and hidden test page `/evil-eye`. Ownership: copied into Brai source.
- `floating-dock.tsx` comes from `@aceternity/floating-dock` (`https://ui.aceternity.com/registry/floating-dock.json`). Status: public source-accessible registry item installed through `shadcn add`. Dependency: existing `motion/react`. Usage: primary fixed app dock navigation. Ownership: copied into Brai source with Brai nav wiring.
- `input-group.tsx` comes from COSS `input-group` (`https://coss.com/ui/r/input-group.json`, dependencies `https://coss.com/ui/r/input.json` and `https://coss.com/ui/r/textarea.json`). Status: public/free registry item, no Pro/paid/gated marker. Dependency: existing `class-variance-authority`. Usage: desktop Actions add field. Ownership: copied into Brai source with local import compatibility.
- `label.tsx`, `separator.tsx`, `field.tsx`, and `progress.tsx` come from the official shadcn CLI commands `npx --yes shadcn@latest add field --overwrite` and `npx --yes shadcn@latest add progress --overwrite`. Status: public/free official shadcn registry source. Dependency: existing `radix-ui`. Usage: Engine OTA progress and shared labeled field primitives. Ownership: copied into Brai source with local import compatibility.
- `switch.tsx` comes from COSS `p-switch-1` dependency source (`https://coss.com/ui/r/p-switch-1.json` and `https://coss.com/ui/r/switch.json`). Status: public/free registry item, no Pro/paid/gated marker. Dependency: existing `@base-ui/react`. Usage: available shared primitive for labeled switches. Ownership: copied into Brai source.
- `checkbox.tsx` comes from COSS `p-checkbox-3` dependencies (`https://coss.com/ui/r/p-checkbox-3.json`, `https://coss.com/ui/r/checkbox.json`, `https://coss.com/ui/r/label.json`). Status: public/free registry items, no Pro/paid/gated marker. Dependency: existing `@base-ui/react`. Usage: Archive action restore checkbox. Ownership: copied into Brai source.
- `markdown-content.tsx` follows the shadcn Typography docs composition (`https://ui.shadcn.com/docs/components/radix/typography`) using existing `react-markdown`. Status: docs composition, not a separate registry component. Usage: reusable Markdown preview rendering with raw HTML skipped.
- `text-effect.tsx` comes from Motion Primitives `text-effect` via the official CLI command `npx motion-primitives@latest add text-effect` and Context7 docs for `per="char"`. Status: public source from Motion Primitives CLI. Dependency: existing `motion`. Usage: per-character page title transitions in `ScreenHeader`. Ownership: copied into Brai source with import/path compatibility only.
- `text-morph.tsx` comes from Motion Primitives `text-morph-button` registry example (`https://raw.githubusercontent.com/ibelick/motion-primitives/main/public/e/text-morph-button.json`). Status: public/free registry source; `@motion-primitives` endpoint returned 429 during install, source copied from upstream registry JSON. Dependency: existing `motion`. Usage: Focus start/stop action label morph. Ownership: copied into Brai source with import/path compatibility only.
- `border-trail.tsx` comes from Motion-Primitives `border-trail` (`https://motion-primitives.com/c/border-trail.json`, mirrored in `https://github.com/ibelick/motion-primitives`). Status: public/free MIT source; shadcn registry endpoint returned 429 during install, source copied from upstream registry JSON. Dependency: existing `motion`. Usage: active Focus timer card border trail. Ownership: copied into Brai source.
- `galaxy.tsx` comes from React Bits `Galaxy-TS-TW` (`https://reactbits.dev/r/Galaxy-TS-TW.json`). Status: public/free registry item, no Pro/paid/gated marker in registry metadata. Dependency: existing `ogl@^1.0.11`. Usage: active Focus timer page background. Ownership: copied into Brai source with import/class compatibility, stable default arrays, and WebGL fallback.
