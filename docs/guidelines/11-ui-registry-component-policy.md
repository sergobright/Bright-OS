# UI Registry Component Policy

- Prefer existing local UI primitives before adding new component code.
- Use only public/free/source-accessible registry items unless explicitly approved otherwise.
- Paid, Pro, license-gated, API-key-gated, private-token-gated, unauthorized, or forbidden registry items are blockers.
- Copy the actual accepted source as the implementation baseline.
- Keep imported source ownership notes in `apps/brai_app/src/shared/ui/README.md`.
- Do not create a visually similar substitute when source is unavailable.
