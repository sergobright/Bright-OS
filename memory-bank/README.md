# Memory Bank

This directory stores public, durable project context for future Brai development sessions.

Read order:

1. `projectbrief.md` - project purpose and scope.
2. `productContext.md` - product behavior and user needs.
3. `activeContext.md` - current focus and immediate next steps.
4. `systemPatterns.md` - architecture and implementation patterns.
5. `techContext.md` - stack, tooling, and constraints.
6. `progress.md` - public baseline progress.
7. `decisionLog.md` - dated durable decisions.

Rules:

- Keep entries public-safe and identity-neutral.
- Put accepted requirements in `openspec/specs/`.
- Put development rules in `docs/guidelines/`.
- Do not store secrets, personal notes, runtime databases, release artifacts, backup names, local home paths, or server-only credentials here.
