# Memory Bank

This directory stores durable project context for future agents and maintainers.

Read order:

1. `projectbrief.md` - project purpose, scope, and success criteria.
2. `productContext.md` - users, needs, and product behavior.
3. `activeContext.md` - current focus, recent changes, and next steps.
4. `systemPatterns.md` - architecture and implementation patterns.
5. `techContext.md` - stack, tooling, commands, and constraints.
6. `progress.md` - completed work, open work, and verification status.
7. `decisionLog.md` - dated project decisions and rationale.

Update rules:

- Record facts discovered from the repository or confirmed by the user.
- Keep placeholders until the information is known.
- Prefer short dated entries for changes and decisions.
- If Memory Bank conflicts with the codebase, trust the codebase and update Memory Bank.
